# Лабораторная работа №4
## Тема: «Парное программирование»
---

# 1 Введение
В рамках лабораторной работы мы реализовали **начальную часть микросервисного онлайн-маркетплейса**, включая систему авторизации, работу с пользователями и базовый интерфейс.  
Документ описывает процесс парного программирования, результаты работы и выводы.

**Технический:**
- **Backend:** Python 3.11 + FastAPI 0.115.0 + Uvicorn + SQLAlchemy 2.0.31 + Alembic + PostgreSQL 17 + Pydantic + JWT (python-jose) + Bcrypt
- **Frontend:** React 18.3.1 + React Router DOM 6.26.0 + Axios 1.7.2 + Vite 5.4.0 (Node.js 18)
- **Инфраструктура:** Docker + docker-compose

# 2 Процесс работы

## 2.1 Выбор стиля ПП
Мы выбрали стиль **"Пинг-понг"**

## 2.2 Этапы разработки

| Итерация | Направление          | Действия |
|:---------|:---------------------|:---------|
| **1**    | Бэкенд → Фронтенд    | • Бэкенд-разработчик создал структуру FastAPI-приложения и эндпоинты `/auth/register`, `/auth/login`<br>• Бэкенд-разработчик реализовал модели SQLAlchemy, схемы Pydantic и хеширование паролей через Bcrypt<br>• Фронтенд-разработчик создал страницы `Login` и `Register` с формами и подключил их к API через Axios |
| **2**    | Фронтенд → Бэкенд    | • Фронтенд-разработчик добавил AuthContext и сохранение JWT-токена<br>• Бэкенд-разработчик реализовал JWT-авторизацию (python-jose), dependency для текущего пользователя и защиту эндпоинтов |
| **3**    | Бэкенд → Фронтенд    | • Бэкенд-разработчик добавил эндпоинты для работы с профилем пользователя (`/users/me`, обновление, удаление)<br>• Фронтенд-разработчик создал `ProtectedRoute` и условное отображение интерфейса в зависимости от наличия токена |
| **4**    | Фронтенд → Бэкенд    | • Фронтенд-разработчик реализовал форму редактирования профиля<br>• Бэкенд-разработчик добавил миграции Alembic, настроил подключение к PostgreSQL и подготовил docker-compose для совместного запуска |

# 3 Результаты работы

## 3.1 Реализованный функционал

| № | Функция                          | Требования |
|---|----------------------------------|------------|
| 1 | Регистрация пользователей        | Создание аккаунта с валидацией и хешированием пароля (Bcrypt) |
| 2 | Авторизация                      | Вход с выдачей JWT-токена |
| 3 | Валидация JWT                    | Проверка токена на бэкенде через dependency |
| 4 | Работа с профилем пользователя   | Получение, обновление и удаление текущего профиля |
| 5 | Защита эндпоинтов                | Ограничение доступа с помощью JWT |
| 6 | Защищённые маршруты на фронтенде | `ProtectedRoute` + AuthContext |
| 7 | Контейнеризация                  | Запуск backend, frontend и PostgreSQL через docker-compose |

## 3.2 Положительные впечатления
1. **Обмен знаниями** — бэкенд-разработчик лучше разобрался в React (Context, Router, Axios), фронтенд-разработчик глубже понял FastAPI, Pydantic и JWT.
2. **Скорость разработки** — благодаря пинг-понгу API сразу работало, не приходилось тратить время на долгие согласования контрактов.
3. **Качество кода** — каждая часть кода проходила двойную проверку сразу в процессе написания.
4. **Меньше багов** — ошибки с токенами, валидацией и CORS находились практически моментально.
5. **Интересный опыт** — работать в паре над полным стеком (Python + React + Docker) оказалось очень увлекательно.

# 4 Проблемы и их решения

| Проблема | Решение |
|:---------|:--------|
| Сложности с настройкой окружения (разные версии Python/Node, порты) | Создали подробный `docker-compose.yml` и инструкцию по локальному запуску |
| CORS-ошибки между фронтендом и бэкендом | Добавили CORS middleware в FastAPI |
| Задачи разного объёма в стиле "пинг-понг" | Разбивали большие задачи на небольшие итерации примерно равной длительности |
| Работа с JWT на фронтенде (хранение, заголовки) | Совместно реализовали AuthContext с автоматическим добавлением токена в запросы |
| "Залипание" и забывание менять роли | Поставили таймер на 25–30 минут для смены ведущего |
